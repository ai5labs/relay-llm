# Supporting Relay

Relay is built and maintained by **ai5labs Research OPC Pvt Ltd** as a fully open-source, Apache-2.0 project. There are several ways to support it — pick the one that fits.

## 1. Contribute code or docs

The fastest way to make Relay better is to fix something that bugs you.

* Browse [open issues](https://github.com/ai5labs/relay-llm/issues) — tagged `good-first-issue` are a solid starting point.
* Add a missing provider, fix a streaming edge case, write a doc page.
* See [CONTRIBUTING.md](CONTRIBUTING.md) for the dev setup, code style, and PR expectations.

You don't need anyone's permission to start. Open a draft PR and we'll work with you.

## 2. Sponsor on GitHub

If your team uses Relay in production and wants to keep it healthy, a small monthly sponsorship goes a long way:

➡️ **[github.com/sponsors/ai5labs](https://github.com/sponsors/ai5labs)**

Every sponsorship dollar pays for:

* Maintainer time on issues, PR review, releases
* Live contract-test budgets (we run Relay against real OpenAI, Anthropic, Groq nightly — that costs money)
* Catalog refresh + new-model day-one support
* CI compute (matrix testing across Python 3.10–3.13 × 3 OSes)

Sponsors are publicly thanked in the README.

## 3. Become a design partner

If you're an enterprise running LLMs in production and want to help shape v0.3 (the proxy mode, multi-tenant primitives, governance UI), we'd love to talk.

**Design partners get:**

* Direct line to the maintainers
* Roadmap influence — we ship the features you actually need
* Free for the design-partner program duration (typically 6–12 months)
* Early access to closed-beta features before they hit OSS

In exchange we ask for:

* Honest feedback (the bugs you hit, the things we got wrong)
* A case study at the end of the engagement (with your sign-off)

📧 Contact us at **engineering@ai5labs.com** with the subject `Relay design partner`.

## 4. Enterprise support contract

Coming with v0.3 (Q3 2026). For companies that want guaranteed response times, custom features, deployment in your VPC, SOC 2 attestation, or BAA / DPA paperwork.

Until then, contact **engineering@ai5labs.com** for a quote and we'll work out a custom arrangement.

## 5. Star the repo + tell people

Genuinely helps a lot at this stage. Star ⭐, share, post about it if you have something to say.

## What we don't take

* **Anonymous donations under $5** — the GitHub processing fee eats them. Don't bother.
* **"Pay to merge"** — we don't take payment to prioritize your PR. We do prioritize design-partner needs in our roadmap, that's different.
* **Crypto** — for legal / compliance reasons.

## Recognition

Sponsors are listed in the README under "Sponsors" alphabetically. Design partners are listed under "Design partners" with their company logo (with their permission). Code contributors show up automatically in GitHub's contributor graph and are linked in release notes.
